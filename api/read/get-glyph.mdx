## Description

Get complete data for a specific glyph including all paths and metrics.

Returns all layers with: paths (nodes with x,y coordinates, type, smooth), components, anchors, width, sidebearings.

Node types: "line" (straight on-curve), "curve" (smooth on-curve), "offcurve" (cubic bezier control point).

Path direction: -1 = counter-clockwise (outer contours), 1 = clockwise (inner contours/counters).

## Parameters

<ParamField path="glyph_name" type="string" required>
  Name of the glyph to retrieve (e.g., "A", "a", "uni0041", "a.ss01")
</ParamField>

## Usage

```python
import anthropic

client = anthropic.Client()

response = client.messages.create(
    model="claude-3-5-sonnet-20241022",
    max_tokens=4096,
    tools=[
        {
            "type": "mcp",
            "server_name": "glyphs-mcp",
            "tool_name": "get_glyph",
            "input_schema": {
                "type": "object",
                "properties": {
                    "glyph_name": {"type": "string"}
                },
                "required": ["glyph_name"]
            }
        }
    ],
    messages=[{"role": "user", "content": "Show me the paths for letter A"}],
)
```

## Notes

- Returns complete path data for all masters
- Node types determine how curves are drawn:
  - `line`: On-curve point with straight segment to previous point
  - `curve`: On-curve point with cubic bezier curve to previous point
  - `offcurve`: Bezier control handle (always comes in pairs before a `curve` node)
- Curve segments consist of: offcurve, offcurve, curve (3 nodes per segment)
- Path direction matters for rendering: outer contours should be counter-clockwise (-1), holes/counters clockwise (1)
- Use this data for analysis or to prepare modifications with `set_glyph_paths()`
