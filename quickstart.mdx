This guide will walk you through your first font analysis using Glyphs MCP.

## Prerequisites

Before you begin, make sure you've completed the [installation](/installation) steps:

- GlyphsApp plugin installed and running
- MCP server connected to your AI assistant
- A font open in GlyphsApp

## Your first analysis

With a font open in GlyphsApp, you can start asking your AI assistant natural language questions.

<Steps>
  <Step title="Check font information">
    Ask your AI assistant to get basic font info:
    
    ```
    What's the font family name and how many glyphs does it have?
    ```
    
    The assistant will use the `get_font_info` tool to retrieve details like family name, UPM, glyph count, masters, and metrics.
  </Step>
  
  <Step title="Analyze stem consistency">
    Check if your stems are consistent across lowercase letters:
    
    ```
    Compare the stems across all my lowercase letters
    ```
    
    This uses the `compare_stems` tool to measure vertical stem thickness across glyphs and identify inconsistencies. Results are automatically marked in GlyphsApp with colors:
    - **Red** = inconsistent
    - **Orange** = unreliable measurement
    - **Yellow** = optical compensation (expected)
    - **Green** = pass
  </Step>
  
  <Step title="Run a color audit">
    Check typographic color (ink density) across your entire font:
    
    ```
    Run a full color audit on my font and tell me which glyphs are inconsistent
    ```
    
    The `audit_font_color` tool analyzes all letter glyphs grouped by category (uppercase, lowercase, figures) and flags glyphs that are too dark or too light compared to industry patterns.
  </Step>
  
  <Step title="Make design changes">
    Ask the assistant to modify a glyph:
    
    ```
    Make the R 20% wider and harmonize the curves
    ```
    
    This will use `rmx_scale` to scale the glyph by 20% in width, then `rmx_harmonize` to optimize the bezier curves for smoothness.
  </Step>
</Steps>

## Common workflows

### Quality control

<AccordionGroup>
  <Accordion title="Check master compatibility">
    Before exporting a multi-master font, verify interpolation compatibility:
    
    ```
    Check if all my glyphs are compatible across masters
    ```
    
    Uses `check_compatibility` to verify paths, nodes, components, and anchors match across all masters.
  </Accordion>
  
  <Accordion title="Analyze kerning quality">
    Review kerning for issues:
    
    ```
    Analyze my kerning and show me any problems
    ```
    
    The `analyze_kerning` tool checks for cross-master gaps, sign changes, outliers, and redundant exceptions.
  </Accordion>
  
  <Accordion title="Validate overshoots">
    Check if round forms overshoot correctly:
    
    ```
    Check overshoot values for all my round glyphs
    ```
    
    Uses `check_overshoots` to verify round and pointed forms overshoot baseline and zone tops by the correct amount (typically 1-2% of zone height).
  </Accordion>
</AccordionGroup>

### Making changes

<AccordionGroup>
  <Accordion title="Adjust glyph weight">
    Make a glyph bolder or lighter:
    
    ```
    Make the R bolder
    ```
    
    Uses `rmx_tune` with a positive weight delta to increase stroke weight using master interpolation.
  </Accordion>
  
  <Accordion title="Create glyph variants">
    Duplicate and modify a glyph:
    
    ```
    Duplicate the 'a' glyph as 'a.ss01' and make it 15% narrower
    ```
    
    Uses `duplicate_glyph` to create a copy, then `rmx_scale` to adjust width.
  </Accordion>
  
  <Accordion title="Harmonize curves">
    Optimize bezier curves after manual editing:
    
    ```
    Harmonize the curves on glyphs H, I, and J
    ```
    
    Uses `rmx_batch` with the harmonize filter to optimize multiple glyphs at once.
  </Accordion>
</AccordionGroup>

## Understanding verdicts

Analysis tools evaluate glyphs and assign verdicts:

| Verdict | Color | Meaning |
| ------- | ----- | ------- |
| **pass** | Green (4) | Glyph meets expected parameters |
| **compensation** | Yellow (3) | Optical compensation detected (expected deviation) |
| **unreliable** | Orange (1) | Measurement can't be reliably determined for this glyph shape |
| **inconsistent** | Red (0) | Glyph fails check — requires attention |

<Info>
  When analyzing all masters, the worst verdict across any master determines the final color marking.
</Info>

## Tips

<Tip>
  When asking for analysis, you can specify which glyphs to check or let the tool analyze all relevant glyphs automatically.
</Tip>

<Tip>
  All write operations are undoable in GlyphsApp with Cmd+Z.
</Tip>

<Warning>
  Some RMX tools require 2+ masters in your font. The `rmx_tune` tool, for example, uses master interpolation and won't work on single-master fonts.
</Warning>
